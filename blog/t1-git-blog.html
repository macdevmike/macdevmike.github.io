When starting out on our programming journey, establishing good habits up front is critical. And learning core concepts like version control, and the tools like Git and GitHub to support the concept are important to learn early. Version control allows a programmer to make consistant saves during development, and encourages experimentation with the code due to the freedom of "rolling back" to a prior version.

Where version control provides the process for saving different versions of your code, Git provides the method. With Git, we have a command line tool and repository system that lets us make incremental saves, and a robust set of tools that allow us to work on individual versions of specific code files. Created by coding legend Linus Torvalds, Git was originally designed for open source coders to work on different "branches" of code for the Linux operating system. Using Git, a programmer can "clone" a piece of code locally, make modifications and "push" these updated "commits" back to the master code branch that is being worked on. As you can see, Git provides a nice way for a group of programmers to coordinate their work on a shared project.

This leaves the final piece: GitHub. As the name implies, GitHub provides a shared social space where programmers can share a "Hub" of code branches. These branches are then merged in a master branch. As the code evolves, each of the developers can fetch the latest changes to the "master branch" which serves as the source of truth in the project.
